# Group

The Group page allows organizations to manage multiple users under a shared subscription tier. From this page, group administrators can view group details, add members, and remove members.

## Understanding Groups

Groups allow organizations to manage multiple users under a shared subscription tier. All members of a group inherit the group's tier, which determines feature availability across the platform. The Group page only appears in your sidebar navigation if you belong to a group.

## Group Information

The top of the page displays the following group details:

| Field | Description |
|-------|-------------|
| **Group Name** | The display name of the group |
| **Group ID** | Unique identifier for the group |
| **Tier** | Subscription tier (Standard, Plus, Premium, Enterprise, or Government) |
| **Subscription Expires** | Expiration date of the group's subscription |
| **Total Seats** | Maximum number of members allowed in the group |
| **Available Seats** | Number of unused seats remaining |
| **Created** | Date the group was created |

## Group Members

The members grid lists all users in the group:

| Column | Description |
|--------|-------------|
| **Username** | Member's username (with an Admin badge for group administrators) |
| **Created** | Account creation date |
| **Actions** | Remove button (group admin only, not shown for other admins) |

## Adding a Member

Group administrators can add new members to the group:

1. Select the `Add User` button above the members grid
2. Enter the username in the side panel form
3. Select `Add User` to confirm

{: .note }
> The `Add User` button is disabled when all seats in the group are occupied. Remove an existing member or contact support to increase your seat count.

## Removing a Member

Group administrators can remove members from the group:

1. Select the remove button in the Actions column for the member
2. Confirm the removal in the dialog

{: .important }
> Removing a user from the group cannot be undone. The removed user will lose access to the group's subscription tier immediately.

{: .note }
> Group administrators cannot be removed from the members list.

## Use Cases

Group management is useful for:

- **Team onboarding** — Adding new team members to share your organization's subscription tier
- **Seat management** — Monitoring available seats and removing inactive users to free up capacity
- **Access control** — Ensuring only authorized users have access to your organization's tier benefits
